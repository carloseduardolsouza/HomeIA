# Guia de Contribuicao

## Branches
Use os prefixos:
- `feature/` para novas funcionalidades
- `fix/` para correcao de bugs
- `chore/` para manutencao
- `docs/` para alteracoes de documentacao

## Commits
Seguimos Conventional Commits:
- `feat:`
- `fix:`
- `docs:`
- `test:`
- `chore:`

## Pull Request
1. Crie branch a partir da `main`.
2. Abra PR com descricao objetiva (contexto, mudanca, impacto).
3. Referencie issue/tarefa associada.
4. Aguarde review e checks do CI antes de merge.

## Checklist de PR
- [ ] Testes passando
- [ ] Lint sem erros
- [ ] Documentacao atualizada
- [ ] Sem segredos/versionamento indevido

## Convencoes de Nome
- Python: `snake_case` para variaveis/funcoes
- JavaScript/TypeScript: `camelCase` para variaveis/funcoes
